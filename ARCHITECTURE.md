# Multi-Device Report Sync Architecture

## Overview
This document describes the architecture for synchronizing and displaying reports from multiple devices running the app on the same local network. The goal is for every device to see all reports generated by any device, in real time or near real time, with no manual configuration.

## 1. Device Discovery
- Use mDNS/NSD (Network Service Discovery) so all devices on the same WiFi network can find each other automatically.
- No manual IP entry required.

## 2. Peer-to-Peer Sync
- Every device runs a lightweight server (e.g., HTTP or TCP socket) to share its reports.
- When a device discovers others, it requests their reports and merges them with its own.
- Each report is treated as a unique record, even if content is identical (no deduplication for now).

## 3. Report Exchange
- On discovery, devices exchange their full report list or just new reports since last sync.
- Reports are merged and stored locally on each device.

## 4. UI
- The Reports tab shows all reports from all devices, grouped by device or time.

## 5. Handling Network Changes
- Devices periodically re-discover peers and re-sync, so new devices joining or leaving is handled automatically.

## 6. Conflict/Duplicate Handling
- For now, no duplicate filtering or conflict resolution is implemented. Every report entry, even if identical, is treated as a unique record. This ensures legitimate repeated entries are always shown.

## Advantages
- No single point of failure (no "main" device needed).
- All devices always have the full report list.
- Works even if some devices go offline temporarily.
